견적 → 거래성사 판정 플로우

이 견적이 실제 거래(출고)로 이어졌는지 자동으로 추정하는 방법

💡
거래내역에는 견적 행과 출고 행이 따로 있을 뿐, 둘을 잇는 번호가 없습니다. 그래서 이렇게 추정합니다 —
“견적을 낸 뒤, 같은 거래처에 같은 품목코드가 실제로 출고됐다면 → 그 견적은 성사된 것.”

▎판정 4단계

1

📦출고 기록을 색인으로 모은다

거래내역 전체에서 ‘출고’ 행만 골라 (거래처 + 품목코드)별로 출고 날짜 목록을 만든다. 공백·하이픈·대소문자 차이는 무시하고 맞춘다.

(우일정공, P50) → [2025-03-12, 2025-08-01, 2026-01-20 …]
2

🔎견적마다 그 색인을 조회한다

견적 한 건(거래처·코드·견적일)에 대해, 같은 (거래처+코드)의 출고 날짜 목록을 찾는다.

3

⚖️기간을 따져 결과를 가른다

견적일 −7일 ~ +365일 사이에 출고가 하나라도 있으면 성사. 없으면, 너무 최근이라 판단이 이른 견적인지 한 번 더 본다.

✅ 성사
1년 내 같은 거래처·코드 출고 있음
⏳ 대기
판단할 후속 판매데이터가 아직 부족한 최근 견적
✗ 미성사
충분한 기간이 지났지만 매칭 출고 없음
4

🖥️견적이력 표에 표시한다

각 견적에 결과 표시(✅/⏳/✗)를 달고, 견적이력 제목 옆에 ‘성사 N / 미성사 N’ 요약을 보여준다.

▎로직 한눈에

성사판정(거래처, 코드, 견적일):
  출고날짜들 = 출고색인[(거래처, 코드)]
  if 출고날짜들 중 (견적일-7일 ~ 견적일+365일)에 하나라도 있음:
    return ✅ 성사
  if (데이터최신일 - 견적일) ≤ 60일:
    return ⏳ 대기  # 판단 보류
  return ✗ 미성사

  • 같은 거래처·코드를 별개 건으로 또 사간 경우도 ‘성사’로 잡힐 수 있음
